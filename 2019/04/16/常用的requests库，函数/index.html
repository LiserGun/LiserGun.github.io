requests库的常用方法

爬虫就是用程序模拟人操作电脑从而来获得信息
然后python自带有urllib库但是用起来。。。很麻烦
所以就有了第三方request库

首先，直接百度python的requests函数都是给你全部而且大部分都是复读机
日常爬虫来说（毕竟我只学到了这里多的还在看）常用的就只有几个

#
requests.get()
requests.post()
以上常用
requests.put()
requests.delete()
requests.head()

其余区别自行百度

以下方法就可以获取url的文本
#
import requests
url = ‘https://www.baidu.com'
get请求
r = requests.get(url)

以下是关于r的各种输出
#
不会输出源代码
print（r）
查看返回的类型，了解一下就行
print(type(r))
查看返回的状态 可以再程序中通过判断是否请求成功从而进行下一步 配合try使用
print(r.status_code)
查看请求头
print(r.headers)
查看cookies 后续可以构建请求头
print(r.cookies)
查看源代码
print(r.text)

网页返回类型是str类型，但是是Json格式可以调用Json方法()来直接解析返回结果
#
r = requests.get(url)
print(r.json())

附加额外信息
例子：https://www.baidu.com?name=Laser&age=19
?后面的参数可以通过自己构建
#
data = {
‘name’:’Laser’,
‘age’:’19’
}
r = requests.get(url,params = data)
输出结果(https://www.baidu.com?name=Laser&age=19)

这样就是一个网页的获取
不过只是单纯的传入url的话
有很多网站并不会获取源码 因为做了爬虫处理
所以就需要用’headers
#
headers = {
‘User-Agent’:’f12查看浏览器即可获得’,
}
r = requests.get(url, headers = headers)
print(r.text)
这样就可以获得很多网站的源码了
如果还不行，那就只有在headers里面添加其余的参数，比如cookies。

在上面讲述的get方法抓取网页中，返回的是一个HTML文档
图片，音频，视频等文件大多数都是以二进制编码所组成的。
如果获得网页上的二进制的话，就可以将他们写成相应的文件

#import requests
r = requests.get('https://github.com/favicon.ico')
with open是打开文件的方式 方法自行百度既有
with open('favicon.ico'(路径),'wb'(####)) as f:
    f.write(写入文件)(f.content二进制的方式) 
    f.write('r.content')

post请求

#    
import requests
data = {'name':'germey','age':'12'}
r = requests.post(url, data = data)

其余方法和上面get一样

文件上传
requests可以模拟网站上传文件
#
import requests
files = {
‘file’:open(‘favicon’, ‘rb’)
}
r = requests.post(url, files = files)

获取cookies
#
import requests
r = requests.get(url)
cookies = r.cookies
输出cookie的名称和值
for key,value in cookies.items():
print(key + ‘: ‘ + value)

会话维持
get和post可以模拟网页请求，但是是2个不同的会话
就相当于使用了2个浏览器打开了不同的页面(而不是一个浏览器中打开2个标签页)
如果2个浏览器加cookie就太麻烦了
毕竟写程序的都喜欢偷懒
所以就有会话维持
import requests
s = requests.Session
s.get(‘http://httpbin.org/cookies/123456')
r = s.get(‘http://httpbin.org/cookies')
